# Dify + Authelia 전체 배포 설정
 
## 구조
 
```
[사용자 브라우저]
        │
        ▼
[Ingress] ← dify.company.com
        │
        ├── /auth/*  → Authelia (로그인/2FA)
        ├── /api/*   → Dify API
        ├── /mcp/* (선택) → mcp-server — 워크플로별 `/mcp/pi`, `/mcp/lims`, `/mcp/edims` (Dify MCP 연동 시)
        └── /*       → Dify Web
       
[내부 연결]
Dify API → Redis, Weaviate, RDS(PostgreSQL), **dify-plugin-daemon** (플러그인·마켓플레이스 등)
dify-plugin-daemon → RDS(PostgreSQL, DB 이름 보통 `dify_plugin`), 로컬/PVC 스토리지 (`/app/storage`)
Authelia → LDAP (회사 기존 서버), RDS(PostgreSQL)
mcp-server → PI Web, Foundry(선택), Oracle(LIMS/EDIMS) 등 (환경변수로 설정)
```
 
### 공개 URL (Authelia·Dify·Ingress 공통)
 
브라우저가 실제로 여는 **호스트·스킴**이 `session.cookies`의 `domain`, `authelia_url`, `access_control.rules[].domain`, Dify의 `CONSOLE_*` / `APP_API_URL` / `FILES_URL`, Ingress의 `host`·`auth-signin`과 **전부 같아야** 합니다.
 
| 용도 | 현재 예시 값 (내부 ELB·HTTP) |
|------|------------------------------|
| 호스트 (FQDN) | `internal-hdo-alb-dev-baollm-pri-ing-1235967427.ap-northeast-2.elb.amazonaws.com` |
| 오리진 | `http://internal-hdo-alb-dev-baollm-pri-ing-1235967427.ap-northeast-2.elb.amazonaws.com` |
 
ELB DNS가 바뀌거나 회사 도메인(예: `https://dify.oilbank.co.kr`)으로 갈 때는 저장소의 `configuration.yml`·본 문서 템플릿·Ingress·Dify 환경변수에서 **동일 문자열을 찾아 한꺼번에** 교체하세요. `*.amazonaws.com` 상위 도메인으로 쿠키를 묶을 수는 없으므로 **항상 실제 FQDN 전체**를 씁니다.

## 2. RDS 준비 (PostgreSQL)
 
기존 RDS에서 실행:
 
```sql
-- Dify용
CREATE DATABASE dify;
CREATE USER dify WITH PASSWORD 'dify-db-password-here';
GRANT ALL PRIVILEGES ON DATABASE dify TO dify;
\c dify
GRANT ALL ON SCHEMA public TO dify;
 
-- Dify 플러그인 데몬용 (메인 Dify DB와 별도 DB 권장)
CREATE DATABASE dify_plugin;
GRANT ALL PRIVILEGES ON DATABASE dify_plugin TO dify;
\c dify_plugin
GRANT ALL ON SCHEMA public TO dify;
\c postgres
 
-- Authelia용 (운영 시, 테스트는 sqlite 사용)
CREATE DATABASE authelia;
CREATE USER authelia WITH PASSWORD 'authelia-db-password-here';
GRANT ALL PRIVILEGES ON DATABASE authelia TO authelia;
\c authelia
GRANT ALL ON SCHEMA public TO authelia;
```

## 3. ConfigMap 목록
 
**Accordion / UI에서 쓰는 방법:** 아래 블록은 **편집 가능한 템플릿**입니다. 플랫폼에 따라 Kubernetes `apiVersion`/`metadata` 헤더는 생략하고, ConfigMap **데이터 키**(`configuration.yml`, `custom.css`, `portal.json` 등)에 **`:` 아래 들여쓴 본문만** 복사해 넣으면 됩니다. `binaryData`(로고·파비콘)는 base64 또는 UI의 파일 업로드로 넣습니다.
 
### ConfigMap 1: `authelia-config`
 
```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: authelia-config
  namespace: dify
data:
  configuration.yml: |
    # ====================
    # 서버 설정
    # ====================
    server:
      address: 'tcp://0.0.0.0:9091'
      asset_path: '/config/assets/'
      path: 'auth'                # /auth subpath
 
      endpoints:
        authz:
          forward-auth:
            implementation: 'ForwardAuth'
 
    theme: 'auto'
 
    # ====================
    # 로그
    # ====================
    log:
      level: 'info'
      format: 'text'
 
    # ====================
    # 2FA - TOTP
    # ====================
    totp:
      issuer: 'YourCompany'       # ← 회사명으로 변경
      period: 30
      skew: 1
 
    # ====================
    # 2FA - WebAuthn (선택)
    # ====================
    # webauthn:
    #   disable: false
    #   display_name: 'YourCompany'
 
    # ====================
    # LDAP 인증
    # ====================
    authentication_backend:
      password_reset:
        disable: true
 
      ldap:
        # ↓↓↓ 회사 LDAP 정보로 변경 ↓↓↓
        address: 'ldap://ldap.company.com:389'
        implementation: 'activedirectory'    # AD면 이거, 일반 LDAP은 'custom'
        timeout: '5s'
        start_tls: false
       
        base_dn: 'dc=company,dc=com'
       
        # 서비스 계정 (LDAP 조회용)
        user: 'cn=authelia,ou=service,dc=company,dc=com'
        password: 'ldap-bind-password'       # ← LDAP 바인드 비밀번호
       
        # 사용자 검색
        users_filter: '(&(|({username_attribute}={input})({mail_attribute}={input}))(objectClass=person))'
        username_attribute: 'sAMAccountName'  # AD면 이거, 일반 LDAP은 'uid'
        mail_attribute: 'mail'
        display_name_attribute: 'displayName'
       
        # 그룹 검색
        # groups_filter: '(&(member={dn})(objectClass=group))'
        # group_name_attribute: 'cn'
        # ↑↑↑ 회사 LDAP 정보로 변경 ↑↑↑
 
    # ====================
    # 세션
    # ====================
    session:
      secret: 'session-secret-minimum-32-characters!!'  # ← 32자 이상 랜덤 문자열
     
      cookies:
        - domain: 'company.com'                          # ← 회사 도메인
          authelia_url: 'https://dify.company.com/auth'
          default_redirection_url: 'https://dify.company.com'
          name: 'authelia_session'
          same_site: 'lax'
          inactivity: '5m'
          expiration: '1h'
          remember_me: '1M'
 
    # ====================
    # Authelia 스토리지(DB) (테스트: sqlite / 운영: postgres)
    # ====================
    # 테스트용 (sqlite)
    storage:
      encryption_key: 'storage-encryption-key-min-32-chars!!'
      local:
        path: '/config/db.sqlite3'
   
    # 운영용 (postgres) - 테스트 후 전환
    # storage:
    #   encryption_key: 'storage-encryption-key-min-32-chars!!'
    #   postgres:
    #     address: 'tcp://your-rds-endpoint:5432'
    #     database: 'authelia'
    #     username: 'authelia'
    #     password: 'authelia-db-password'
 
    # ====================
    # 알림 (테스트: 파일 / 운영: SMTP)
    # ====================
    # 테스트용
    notifier:
      filesystem:
        filename: '/config/notification.txt'
   
    # 운영용 (SMTP) - 테스트 후 전환
    # notifier:
    #   smtp:
    #     address: 'smtp://mail.company.com:587'
    #     username: 'authelia@company.com'
    #     password: 'smtp-password'
    #     sender: 'Authelia <authelia@company.com>'
 
    # ====================
    # 접근 제어
    # ====================
    access_control:
      default_policy: 'two_factor'
     
      rules:
        # Authelia 자체 리소스
        - domain: 'dify.company.com'
          resources:
            - '^/auth/api/.*$'
            - '^/auth/.*$'
          policy: 'bypass'
       
        # 사내망 1FA - 사내망에서 2FA 패스할경우 회사 ip설정
        - domain: 'dify.company.com'
          networks:
            - '10.0.0.0/8'
            - '192.168.0.0/16'
          policy: 'one_factor'
       
        # 외부 2FA
        - domain: 'dify.company.com'
          policy: 'two_factor'
 
    # ====================
    # OIDC Provider (for Dify SSO)
    # ====================
    #
    # Generate client_secret hash (any tool that produces argon2id):
    #
    # [Python] pip install argon2-cffi
    #   python -c "from argon2 import PasswordHasher; print(PasswordHasher().hash('your-plain-secret'))"
    #
    # [Node.js] npm install argon2
    #   node -e "require('argon2').hash('your-plain-secret').then(console.log)"
    #
    # [Linux] echo -n 'your-plain-secret' | argon2 "$(openssl rand -base64 16)" -id -e
    #
    # Use the plain secret in dify-sso .env (OIDC_CLIENT_SECRET)
    # Use the generated hash below in client_secret
    identity_providers:
      oidc:
        clients:
          - client_id: 'dify'
            client_name: 'Dify'
            client_secret: '$argon2id$v=19$m=65536,t=3,p=4$...'  # ← Replace with generated hash
            public: false
            authorization_policy: 'two_factor'    # or 'one_factor'
            redirect_uris:
              - 'https://dify.company.com/console/api/enterprise/sso/oidc/callback'
            scopes:
              - 'openid'
              - 'profile'
              - 'email'
            response_types:
              - 'code'
            token_endpoint_auth_method: 'client_secret_post'
            pkce_challenge_method: 'S256'
 
```

### 파드 4: dify-api

|------|-----|
| 이미지 | `langgenius/dify-api:latest` |
| 포트 | 5001 |
 
환경변수:
```
# 모드
MODE=api
 
# 시크릿 키 (32자 이상)
SECRET_KEY=sk-your-secret-key-minimum-32-chars!!
 
# PostgreSQL (RDS)

DB_HOST=your-rds-endpoint.ap-northeast-2.rds.amazonaws.com
DB_PORT=5432
DB_DATABASE=dify
 
# Redis
REDIS_HOST=redis
REDIS_PORT=6379
REDIS_PASSWORD=difypassword
 
# Weaviate
VECTOR_STORE=weaviate

# 콘솔 URL
CONSOLE_WEB_URL=https://dify.company.com
CONSOLE_API_URL=https://dify.company.com
 
# 파일 업로드
```

### 파드 6: dify-plugin-daemon
 
Dify **플러그인 / 마켓플레이스** 등을 쓰려면 API·워커와 별도로 띄우는 서비스입니다. **Ingress로 노출할 필요 없음** — 클러스터 내부에서 `dify-api`만 `PLUGIN_DAEMON_URL`로 호출합니다.
 
| 항목 | 값 |
|------|-----|
| 이미지 | `langgenius/dify-plugin-daemon:<버전>-local` (`-local` 이미지에 플러그인 런타임 포함. 사용 중인 `dify-api` 이미지 태그에 맞춰 저장소 `docker/docker-compose.yaml`의 `plugin_daemon` 항목 확인) |
| 포트 | **5002** — 메인 HTTP (`SERVER_PORT`). **5003** — 원격 설치·디버그(`PLUGIN_REMOTE_INSTALLING_*`); 개발·디버깅 시에만 Service/Ingress로 열고 운영에서는 막는 것을 권장 |
 
**PostgreSQL**: RDS에 `dify_plugin` DB 생성(2절 SQL) 후, 데몬은 메인 Dify와 동일 호스트·계정을 쓰고 **`DB_DATABASE=dify_plugin`** 만 다르게 둡니다.
 
**스토리지**: 공식 구성은 `/app/storage` 에 플러그인 작업 디렉터리·캐시를 둡니다. **NFS 등 네트워크 볼륨은 비권장**(데몬 README — 공유 작업 경로 특성). 가능하면 **노드 로컬 디스크 기반 PVC** 또는 `emptyDir`(비운영)을 사용하세요.
 
환경변수 (핵심만; 객체 스토리지·S3 등은 [공식 compose](https://github.com/langgenius/dify/blob/main/docker/docker-compose.yaml) `plugin_daemon` 블록 참고):

### 파드 8: mcp-server (FastMCP / Streamable HTTP)
 
| 항목 | 값 |
|------|-----|
| 이미지 | MCP 서버 Docker 이미지 (`Dockerfile` 빌드) |
| 포트 | 8000 |
| 헬스 | `GET /health` → `{"status":"ok"}` |
 
**MCP (Streamable HTTP)**: 동일 파드에서 워크플로별로 경로가 나뉩니다. Dify·클라이언트는 필요한 API에 맞는 URL만 등록합니다.
 
| 경로 | 도구 그룹 |
|------|-----------|
| `/mcp/pi` | PI Web API |
| `/mcp/lims` | LIMS Oracle + Foundry(`lab_result_meta_search`) |
| `/mcp/edims` | EDIMS DB·문서·파일 검색·추출 |
 
Ingress에서 `path: /mcp`, `pathType: Prefix`이면 위 하위 경로가 모두 mcp-server로 전달됩니다.
 
**Oracle Instant Client:** 이미지 **빌드 시** `Dockerfile`에서 Linux x86-64 Basic Light ZIP을 `wget`으로 받아 `/opt/oracle/instantclient`에 풀어 넣습니다. **배포 파드에는 별도 설치 스크립트나 IC 볼륨이 필요 없고**, 아래 **환경변수만** 맞추면 됩니다.
 
| 변수 | 설명 |
|------|------|
| `EDIMS_ORACLE_USER` | EDIMS 계정 |
| `EDIMS_ORACLE_PASSWORD` | Secret 권장 |
| `EDIMS_ORACLE_DSN` | `host:port/SERVICE` 또는 `host:port:SID` (JDBC URL 통째로 넣지 않음) |
| `EDIMS_ORACLE_CLIENT_LIB_DIR` | **비우면** 이미지 기본 `/opt/oracle/instantclient`(thick). **thin만** 쓰려면 빈 값으로 명시해 덮어쓰기 |
| `EDIMS_ORACLE_SCHEMA` | 기본 `HDOEDIMS` |
| `EDIMS_ORACLE_CONNECT_TIMEOUT` | 기본 30 |
| `EDIMS_ORACLE_MAX_ROWS` | 기본 500 |
 
**LIMS Oracle (thin, Instant Client 불필요)**
 
| 변수 | 설명 |
|------|------|
| `LIMS_ORACLE_USER` | 사용 시 필수 |
| `LIMS_ORACLE_PASSWORD` | Secret |
| `LIMS_ORACLE_DSN` | 위와 동일 형식 |
| `LIMS_ORACLE_CONNECT_TIMEOUT` | 기본 30 |
| `LIMS_ORACLE_MAX_ROWS` | 기본 5000 |
 
**PI Web (필수)**
 
| 변수 | 설명 |
|------|------|
| `PIWEB_BASE_URL` | 예: `https://host/piwebapi` |
| `PIWEB_USERNAME` | |
| `PIWEB_PASSWORD` | Secret |
| `PIWEB_HDO_WEB_ID` | |
| `PIWEB_HDC_WEB_ID` | 없으면 비움 |
 
**기타 (선택·기본값은 `.env.example`·README 참고)**
 
`CORE_FOUNDRY_*`, `EDIMS_SFTP_*`, `EDIMS_LIBREOFFICE_PATH`, `EDIMS_DWG_*`, `EDIMS_MR_*`, `OCR_LANG`, `TESSERACT_CMD`, `MCP_HOST`, `MCP_PORT`, `LOG_LEVEL` 등 — 앱 설정과 동일한 **대문자 스네이크** 환경변수명을 사용합니다.

## 6. Ingress 설정
 
```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: dify-ingress
  namespace: dify
  annotations:
    kubernetes.io/ingress.class: nginx
   
    # TLS 설정 (cert-manager 사용 시)
    cert-manager.io/cluster-issuer: letsencrypt-prod
   
    # Authelia 인증 연동
    nginx.ingress.kubernetes.io/auth-url: "http://authelia.dify.svc.cluster.local:9091/api/verify"
    nginx.ingress.kubernetes.io/auth-signin: "https://dify.company.com/auth"

    # /auth 경로는 인증 체크 제외
    nginx.ingress.kubernetes.io/auth-snippet: |
      if ($request_uri ~ ^/auth) {
        return 200;
      }
spec:
  tls:
    - hosts:
        - dify.company.com
      secretName: dify-tls
  rules:
    - host: dify.company.com
      http:
        paths:
          # Authelia (로그인 화면)
          - path: /auth
            pathType: Prefix
            backend:
              service:
                name: authelia
                port:
                  number: 9091

          - path: /api
            pathType: Prefix
            backend:
              service:
                name: dify-api
                port:
                  number: 5001

            pathType: Prefix
            backend:
              service:
                name: dify-api
                port:
                  number: 5001
         
          - path: /files
            pathType: Prefix
            backend:
              service:
                name: dify-api
                port:
                  number: 5001
         
          - path: /v1
            pathType: Prefix
            backend:
              service:
                name: dify-api
                port:
                  number: 5001
         
          # MCP (Dify MCP 도구 연동 시). Prefix /mcp 로 /mcp/pi, /mcp/lims, /mcp/edims 까지 동일 서비스로 라우팅.
          # 경로 리라이트는 Ingress 컨트롤러에 맞게 조정.
```

**Windows에서 OpenSSL 없이 난수 만들기 (PowerShell)**  
32바이트를 hex로 출력하면 64글자가 됩니다. `session.secret`, `storage.encryption_key`, `SECRET_KEY` 등 각각 **별도로** 실행해 서로 다른 값을 쓰세요.
 
```powershell
$b = New-Object byte[] 32
[System.Security.Cryptography.RandomNumberGenerator]::Create().GetBytes($b)
-join ($b | ForEach-Object { $_.ToString('x2') })
```
 
한 줄로 실행:
 
```powershell
& { $b = New-Object byte[] 32; [System.Security.Cryptography.RandomNumberGenerator]::Create().GetBytes($b); -join ($b | ForEach-Object { $_.ToString('x2') }) }
```
 
PowerShell 7 이상:
 
```powershell
$b = [byte[]]::new(32)
[System.Security.Cryptography.RandomNumberGenerator]::Fill($b)
-join ($b | ForEach-Object { $_.ToString('x2') })
```
 
OpenSSL을 쓸 수 있으면 `openssl rand -hex 32` 와 동일한 길이입니다. 필요 시 `winget install ShiningLight.OpenSSL.Light` 등으로 설치할 수 있습니다.
